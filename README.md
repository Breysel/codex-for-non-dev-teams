# Codex for Non-Developer Teams

A lightweight HTML enablement kit for helping non-developer teams understand where Codex is useful: slide creation, interactive pages, data analysis, reporting, workflow checks, and safe automation.

## Pages

- `index.html` - overview hub
- `codex-one-pager.html` - internal pitch
- `facilitator-deck.html` - 10-slide workshop deck
- `codex-best-practices.html` - practical habits and prompt starters
- `who-we-are.html` - Stage+ team context
- `codex-vp-pitch.html` - source for the condensed VP pitch deck
- `codex-for-non-dev-teams-vp-pitch.pdf` - 5-slide pitch deck for stakeholder sharing
- `codex-complete-kit.html` - combined source for the complete PDF kit
- `codex-for-non-dev-teams-complete-kit.pdf` - one forwardable PDF with section jump links

## Suggested Use

Open `index.html` locally, or publish the repo with GitHub Pages from the repository root.

For stakeholder sharing, GitHub Pages is the simplest web option because this is a static site. After enabling Pages for the repository, the share URL should be:

`https://breysel.github.io/codex-for-non-dev-teams/`

This is a draft enablement kit, not official OpenAI or Universal Music documentation.
